# Sessão: 2026-06-20 — Fix do /update + Setup da Wiki Persistente

## 1. Fix definitivo do /update

O `/update` estava destruindo sessões, skills e config a cada execução.

**Causa raiz:** o agente havia modificado 3 skills bundled (git-tracked) em vez de criar arquivos novos. Isso causava `M` no git status → `git stash push --include-untracked` → conflito no stash apply → `git reset --hard HEAD` apagava todos os untracked (dados do usuário).

**Fixes aplicados:**

| Fix | Arquivo | O que faz |
|---|---|---|
| `.git/info/exclude` | `/root/.hermes/.git/info/exclude` | Exclui arquivos críticos de runtime do stash permanentemente |
| Config | `config.yaml` | `non_interactive_local_changes: discard` → `stash` |
| Regra | `SOUL.md` | Proíbe editar bundled skills; exige `git ls-files` antes |

**Skills bundled revertidas** (tinham sido modificadas incorretamente):
- `skills/note-taking/obsidian/SKILL.md`
- `skills/research/llm-wiki/SKILL.md`
- `skills/github/github-repo-management/SKILL.md`

**User skills criadas** (conteúdo salvo corretamente):
- `skills/note-taking/obsidian-ai-memory/SKILL.md`
- `skills/research/llm-wiki-ai-memory/SKILL.md`
- `skills/github/github-token-masking/SKILL.md`

Hermes: v0.16.0 → v0.17.0. 141 sessões intactas.

## 2. Setup da wiki persistente

Migrado de ai-memory (Basic Memory, Docker) para wiki Git pura.

- Wiki clonada em `/root/wiki/` (repo: `omgiova/ai-memory-wiki`)
- Hook `post-commit`: `git pull --rebase && git push` automático após cada commit
- `SOUL.md` atualizado: Hermes sabe onde fica a wiki e como usá-la
- ai-memory: Docker parado, MCP desabilitado no `config.yaml`

## Lições

- **Bundled skill** = rastreada por git (`git -C /root/.hermes ls-files skills/...` retorna path). PROIBIDO editar.
- **User skill** = não rastreada (retorna vazio). Pode editar livremente.
- `/update` usa `git stash push --include-untracked` — `.git/info/exclude` é a proteção correta (local, nunca commitado, permanente).
